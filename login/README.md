# 로그인 처리1 - 쿠키, 세션

> 인프런 **김영한의 스프링 MVC 2편 - 백엔드 웹 개발 활용 기술** Section 7 강의를 학습하며 작성한 코드 리포지토리입니다.

## 학습 내용

### 쿠키를 이용한 로그인 처리
- 로그인 성공 시 HTTP 쿠키를 생성해 클라이언트에 전달하는 방식으로 로그인 상태를 유지
- `Cookie` 객체를 직접 생성하고 `HttpServletResponse`에 추가하는 방법 학습
- 쿠키의 보안 문제점 파악 (클라이언트 측 위변조 가능, 중요 정보 노출 위험)

### 세션을 이용한 로그인 처리
- 쿠키의 보안 문제를 해결하기 위해 서버에서 세션을 직접 구현하는 방법 학습
- UUID 기반의 세션 ID를 생성하고 서버 메모리(`Map`)에 세션 데이터를 보관하는 방식으로 동작 원리 이해
- 세션 생성, 조회, 만료 기능을 직접 구현한 `SessionManager` 작성

### HttpSession
- 서블릿이 기본 제공하는 `HttpSession`을 활용한 로그인 처리 방법 학습
- `SessionConst`를 통한 세션 키 상수화 및 `@SessionAttribute` 어노테이션으로 코드 간소화
- 세션 타임아웃 설정 (`session.setMaxInactiveInterval`) 및 LastAccessedTime 기반 세션 만료 방식 이해

## 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| 쿠키 | 클라이언트(브라우저)에 저장되는 데이터. 위변조 위험 존재 |
| 세션 | 중요 데이터를 서버에 저장하고 클라이언트에는 세션 ID(UUID)만 전달 |
| `HttpSession` | 서블릿이 제공하는 세션 기능. 직접 구현 없이 안전한 세션 관리 가능 |
| `@SessionAttribute` | 스프링이 제공하는 어노테이션. 세션에서 데이터를 편리하게 조회 |
| 세션 타임아웃 | 일정 시간 요청이 없으면 세션 자동 만료. 메모리 낭비 방지 |
